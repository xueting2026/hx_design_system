# AGENTS.md — AI 消费指引

**你是 AI agent，要用信小递设计系统做界面原型？先读完这页再动手。**

信小递是海信集团物流司机端微信小程序的设计系统。设计性格：简洁实用、卡片圆角、克制色彩、清晰层级、操作效率优先。品牌色 `#00AAA6`，设计宽度 375px。

## 一条铁律

**值只从 `tokens.json` 取，颜色只用调色板，字体只用 PingFang SC，间距/圆角/控件尺寸只用既定档位——绝不自创值、自创组件、随意加装饰。**

## 🚨 组件复用铁律（最高优先级）

**所有页面必须引用 `xinxiaodi-ui.css` 并使用其中定义的 CSS class 来构建组件。**

- preview/ 和 examples/ 中的组件是经过精心调教的标准实现
- 生成 demo 或页面时，**禁止用纯内联 style 重新实现任何已有组件**
- 如果某个 UI 元素在 xinxiaodi-ui.css 中有对应 class，必须使用该 class
- 仅对组件 JSON 中定义的「card 内部结构」等页面特定布局可使用内联样式补充
- **不确定用什么组件 → 必须询问用户，不允许自行发挥**

违反此规则 = 交付失败，必须返工。

## 消费顺序

1. **取值** → `tokens.json`：颜色、字体、间距、圆角、阴影、组件尺寸、页型定义。**直接引用，不改不近似。**
2. **取规则** → `xinxiaodi-Design.md`：完整设计规范（概述 → 颜色 → 排版 → 布局 → 组件 → 按钮组 → 图标 → 页型 → 该做与不该做）。
3. **看组件** → `preview/` 目录下的独立组件预览页：每个组件的视觉和交互。
4. **取素材** → `assets/`：SVG 图标、Logo、金刚区图标，按命名规则引用。
5. **看成品** → `examples/` 目录下的页型模板示例（首页、列表页、交互 demo）。

## 按任务怎么做

### ⚠️ 生成页面前必须先问

在生成任何 demo/页面之前，**必须先询问用户**：
> "这个页面是做 demo 演示用，还是直接交付开发？"

- **demo 演示** → 使用 375px 固定宽度，保持现有 CSS 不变
- **交付开发** → 改为响应式写法：宽度用 `100% + max-width: 750px`，内容区用 `calc(100% - 32px)`，其他高度/间距保持 px（开发时 px×2=rpx）

### 做列表页

信小递最核心的页型。结构从上到下：
```
状态栏(44px) → 标题栏(44px) → 定位组件(48px,可选) → 扫码接单(48px,可选) → 搜索框(36px) → Tab(30px,可选) → 卡片列表(间距12px) → 浮动按钮(可选)
```

参考 `examples/demo-list.html`。

### 做操作页

结构：标题栏 → 数据展示区（只读）→ 录入区（输入/选择/上传）→ 底部固定提交按钮(40px)。

### 做弹窗

宽度固定 320px，内边距 24px。右上角关闭按钮(24px, 绝对定位 top 8 right 8)。按钮组：浅色在左 + 深色在右（双按钮）或仅深色（单按钮）。蒙层 rgba(0,0,0,0.5)。

### 做首页

渐变顶部(194px) + 扫码接单入口 + 金刚区(48px 图标网格) + 待办快捷区 + 浮动按钮。参考 `examples/demo-home.html`。

## 图标引用

```html
<img src="assets/icons/{功能}_{颜色}_{尺寸}px.svg" style="width:{尺寸}px; height:{尺寸}px;" />
```

尺寸档：48px（金刚区）/ 24px（通用）/ 20px（工具）/ 16px（辅助）/ 12px（箭头）

颜色变体：gray / green / white / black

示例：
- `search_gray_24px.svg` — 搜索图标灰色 24px
- `position_green_16px.svg` — 定位图标绿色 16px  
- `select/down_surface_green_12px.svg` — 下拉箭头绿色填充 12px
- `close_gray_24px.svg` — 关闭图标灰色 24px

金刚区：`assets/shortcuts/icon/home/driver/{功能}.svg`

## 按钮组规范（卡片底部）

```
有「x单x件」→ 右侧按钮 ≤ 3个
无统计文字 → 右侧按钮 ≤ 5个
超出时 → 显示「更多」文字按钮(14px, #00AAA6, 无底色)
```

「更多」点击弹出浮窗：
- 白色背景、8px 圆角
- 阴影 `0 4px 16px rgba(0,0,0,0.12)`
- 选项 padding 8px 16px，14px Regular #1D2129

## 产出前自查清单（最常违反的）

### 🚨 硬规则（出现即失败，必须修正后才能交付）

- [ ] **必须引用 xinxiaodi-ui.css** — `<link rel="stylesheet" href="../design-system/xinxiaodi-ui.css">`，不允许不引用
- [ ] **必须使用 CSS class** — 按钮用 `.btn .btn--sm .btn--primary`，不允许纯内联样式写组件
- [ ] **禁止自创组件** — 所有组件必须复用 xinxiaodi-ui.css 中已定义的 class（.searchbox / .tag / .card / .location-bar / .tab-item / .dialog / .btn-group 等）。如果不确定用什么，必须询问用户
- [ ] **图标** — 必须引用 `assets/icons/` 下的 SVG 文件，不自创内联 SVG。缺少图标时告知用户
- [ ] **颜色** — 全部来自 token 调色板或 CSS 变量 (var(--vi-2) 等)，无裸 HEX 值
- [ ] **间距**：全部是 4 的倍数（4/8/12/16/24/32）
- [ ] **圆角**：只用 2/4/10/12/50% 这几个档
- [ ] **字体**：只用 PingFang SC，按钮文字必须 Regular(400)
- [ ] **页面宽度**：demo 模式用 375px 固定宽；开发交付模式用 100% + max-width: 750px 响应式
- [ ] **标签颜色**：蓝=运单/提货，橙=2C/收货，紫=电商补货
- [ ] **禁用态**：只降文字 opacity 40%，不改背景色
- [ ] **placeholder**：用 text-1(#BFC5CC)，不用 text-2
- [ ] **bg-1 用对**：卡片内嵌区域用 #F3F5F7，不用白色
- [ ] **按钮组**：一组中只突出一个主操作样式
- [ ] **弹窗**：宽 320px，关闭按钮右上角，按钮浅色在左深色在右
- [ ] **搜索框**：高 36px，查询按钮 28px 高 #00AAA6 背景白字
- [ ] **Tab**：选中态 #00AAA6 背景白字，未选中白底黑字

## 常见错误

❌ 把 `bg-2(#EBEDF0)` 当通用背景 → ✅ 只用于预改约底部
❌ 按钮用 Medium/Semibold 字重 → ✅ 按钮统一 Regular(400)
❌ 搜索框 placeholder 用 #7C8492 → ✅ 用 #BFC5CC(text-1)
❌ 标签颜色随意选 → ✅ 蓝/橙/紫各有固定业务含义
❌ 浮动按钮放页面中间 → ✅ 只放右下角固定定位
❌ 卡片内嵌区用白色底 → ✅ 用 bg-1(#F3F5F7) 区分层级
❌ 用奇数间距(3px/7px/13px) → ✅ 只用 4 的倍数
❌ 圆角混用(有 6px 有 10px 有 8px) → ✅ 同一视图用统一圆角家族
