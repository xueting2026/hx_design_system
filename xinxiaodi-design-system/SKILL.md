# 信小递设计系统 Skill

读取本目录下的文件，按照信小递设计规范产出 HTML 原型或组件。

如果用户没有明确指示，问他们要做哪种页面（列表页/操作页/首页/登录页/弹窗），然后按规范输出。

## 快速索引

- `SKILL.md` — 本文件，品牌速览（先读）
- `tokens.json` — 设计 Token 机读单一事实源（颜色/字体/间距/圆角/组件尺寸）
- `xinxiaodi-Design.md` — 完整设计规范（八大章节 + 页型决策树 + 该做与不该做）
- `assets/` — SVG 图标、Logo、金刚区图标

## 品牌一览

- **平台**：微信小程序（移动端 375px）
- **品牌色**：`#00AAA6`（信小递绿）——按钮、链接、选中态、Tab 激活
- **设计性格**：简洁实用、卡片圆角、克制色彩、清晰层级、操作效率优先
- **字体**：PingFang SC（唯一字族，禁衬线）
- **圆角**：2px 标签 / 4px 按钮输入框 / 10px 卡片 / 50% 头像浮窗

## 核心约束（铁规）

1. **品牌色 `#00AAA6` 只用于**：按钮填充、链接文字、选中态、Tab 激活态背景、图标着色
2. **图标必须引用 assets**：使用 `assets/icons/` 下的 SVG 文件，不要自创内联 SVG 图标。如果缺少某个图标或不确定用哪个，请明确告知用户
3. **必须复用 design-system 中已有的组件**：所有组件（按钮、搜索框、标签、卡片、Tab、弹窗、导航等）必须引用 `xinxiaodi-ui.css` 中定义的 class。preview/ 和 examples/ 中的组件都是经过精心调教的，**禁止自己重新生成组件样式**。如果不确定用什么组件，必须询问用户
4. **禁用态** = 背景不变 + 文字/图标 opacity 40%
3. **hover 态** = `#009093`（vi-3），按下缩小 scale(0.97)
4. **间距只用 4 的倍数**：4/8/12/16/24/32px
5. **圆角不混用**：同一视图内保持统一圆角家族
6. **标签颜色承载业务含义**：蓝=运单状态/提货点，橙=2C/收货点，紫=电商补货——不可随意替换
7. **按钮文字统一 Regular**，不用 Medium/Semibold
8. **卡片内嵌内容用 `bg-1` 底色**，不用白色（区分层级）
9. **搜索框 placeholder 用 `text-1`（#BFC5CC）**，不用 `text-2`
10. **按钮组规范**：有统计文字时右侧≤3个按钮，无统计时≤5个，超出显示「更多」

## 六种页型

```
用户要做什么？
├── 认证身份 → 登录页
├── 概览全局 → 首页
├── 选子功能(≥3个) → 功能选择页
├── 浏览/筛选/操作列表 → 列表页（最常用）
├── 录入信息并提交
│   ├── 复杂(多字段/上传) → 操作页
│   └── 简单(1-2个字段) → 弹窗/面板
└── 查看详情 → 详情页
```

## 列表页组件构成

列表页是信小递最核心的页型，标准构成从上到下：

1. 状态栏 (44px)
2. 标题栏 (44px, 返回箭头 + 标题 + 小程序胶囊)
3. 定位组件 (48px, 可选)
4. 扫码接单入口 (48px, 可选)
5. 搜索框 (36px, 可附加筛选/排序)
6. Tab 切换 (30px, 可选)
7. 卡片列表 (间距 12px)
8. 浮动按钮 (可选)

## 做界面时

1. 从 `tokens.json` 取值，不自创颜色/字号/间距
2. 组件样式参考 `preview/` 下的独立组件预览页
3. 页面布局遵循 `信小递Design.md` 中的页型规范
4. 图标从 `assets/icons/` 引用 SVG
5. 设计宽度固定 375px（移动端小程序）

## 产出前自查

- [ ] 颜色全部来自 token，无裸 HEX
- [ ] 间距全部是 4 的倍数
- [ ] 圆角用对应档位（2/4/10/12/50%）
- [ ] 标签颜色匹配业务含义
- [ ] 按钮文字用 Regular 字重
- [ ] 禁用态只降透明度不改背景
- [ ] 一组操作中只突出一个主按钮
- [ ] placeholder 用 text-1 色
- [ ] 页面宽度 375px，内容宽度 343px（两侧各 16px）
- [ ] 如果是开发交付，改用响应式写法（100% + max-width: 750px）
